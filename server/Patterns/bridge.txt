PADRÃO BRIDGE (Estrutural)

CONTEXTO NO PROJETO:
- Separação entre abstração e implementação de funcionalidades
- Diferentes implementações de autenticação (JWT, OAuth, etc.)
- Múltiplos provedores de storage (Cloudinary, AWS S3, local)

APLICAÇÃO PRÁTICA:
1. AuthBridge - abstração para diferentes métodos de autenticação
2. StorageBridge - interface comum para diferentes provedores de storage
3. NotificationBridge - abstração para email, SMS, push notifications
4. ReportBridge - diferentes formatos de relatório (PDF, CSV, JSON)
5. DatabaseBridge - abstração para diferentes bancos de dados

BENEFÍCIOS:
- Permite trocar implementações sem afetar o código cliente
- Extensibilidade para novos provedores
- Separação clara entre interface e implementação
