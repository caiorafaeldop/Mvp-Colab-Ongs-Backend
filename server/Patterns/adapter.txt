PADRÃO ADAPTER (Estrutural)

CONTEXTO NO PROJETO:
- Integração com APIs externas (Cloudinary, OpenAI, Anthropic)
- Adaptação de diferentes formatos de dados entre camadas
- Conversão de respostas de LLM para formato padronizado

APLICAÇÃO PRÁTICA:
1. CloudinaryAdapter - adapta interface do Cloudinary para o domínio
2. LLMAdapter - unifica diferentes provedores de IA (OpenAI, Anthropic)
3. DatabaseAdapter - adapta MongoDB para repositórios
4. AuthAdapter - adapta JWT para diferentes estratégias de auth

BENEFÍCIOS:
- Desacoplamento entre sistemas externos e domínio
- Facilita troca de provedores
- Padronização de interfaces
